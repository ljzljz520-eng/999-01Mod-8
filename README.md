# FAQuery - 固定资产编码查询系统

FAQuery 是一个现代化的 Web 应用程序，用于通过服务器 IP 和固定资产编码 (FACode) 快速查询对应的序列号 (SN)。系统采用前后端分离架构，完全容器化部署。

## ✨ 主要特性 (V8 Release - 权限分级)

*   **🔐 三级权限体系**: 基于 RBAC 的细粒度访问控制
    - **普通员工**: 只能查询自己领用或本部门的资产，敏感字段（采购价格）自动过滤
    - **资产管理员**: 可查看全量资产数据，支持导出、生成维修链接
    - **外包维修人员**: 仅能通过一次性链接查看维修相关字段
*   **📝 越权审计**: 所有越权尝试自动记录 IP 地址、账号、操作类型、原因
*   **📤 导出鉴权**: 导出功能后端强制校验，非管理员直接拒绝（不仅仅是前端隐藏按钮）
*   **🔗 一次性链接**: 24小时有效，单次使用，标记访问时间和访问者
*   **🛡️ 字段级权限**: 根据角色动态过滤返回字段
*   **纯 MySQL 架构**: 内置 MySQL 8.0 容器，开箱即用，自动初始化测试数据。
*   **多环境支持**: 支持配置多个 MySQL 数据库连接（生产/测试环境），并在界面上一键切换。
*   **现代化 UI**: 响应式设计，渐变背景，全中文界面，极致的用户体验。
*   **全面校验**: 前端对所有输入进行严格校验，防止无效请求。
*   **智能错误翻译**: 自动将底层数据库错误翻译为友好的中文提示。
*   **完全容器化**: 基于 Docker Compose 的一键部署。

## 🛠️ 技术栈

*   **前端**: HTML5, Vanilla JavaScript (ES6+), Tailwind CSS (无构建依赖模式)

## 🚀 快速开始

### 前提条件
*   已安装 Docker 和 Docker Compose。

### 部署步骤

1.  **克隆项目**
    ```bash
    git clone <repository-url>
    cd FAQuery
    ```

2.  **启动服务**
    ```bash
    docker compose up -d
    ```
    > 首次启动时，MySQL 容器会自动执行初始化脚本 (`mysql-init/init.sql`) 创建表和用户。

3.  **访问应用**
    打开浏览器访问: `http://localhost:3000`

### 使用指南

#### 1. 登录系统
首次访问会显示登录界面，使用测试账号登录：
*   **管理员账号**: `wangwu` / `123456`
*   **普通员工账号**: `zhangsan` / `123456`

#### 2. 资产查询
登录后可进行查询操作：
*   **服务器 IP 或域名**: `localhost`
*   **固定资产编码**: `FA001`
*   **预期结果**: 完整的资产信息（根据角色显示不同字段）

#### 3. 数据权限说明
| 角色 | 查询范围 | 可见字段 |
|------|---------|---------|
| 普通员工 | 本人领用 + 本部门资产 | 排除 `purchase_price` 等敏感字段 |
| 管理员 | 全部资产 | 所有字段 |
| 外包维修 | 仅通过一次性链接 | 仅维修相关字段 |

#### 4. 导出数据
仅管理员角色可见「导出数据」按钮，点击后下载全量 CSV 数据。后端强制鉴权，即使绕过前端也无法导出。

#### 5. 生成维修链接
管理员查询资产后可点击「生成维修链接」，系统会生成一个 24 小时有效的一次性链接，外包人员打开后只能看到维修相关字段。

#### 6. 添加外部数据库连接
点击右上角的 **齿轮图标** 进入“数据库连接管理”：
1.  点击 **+ 新增 MySQL 连接**。
2.  填写连接信息（支持粘贴 `mysql://` 格式的连接字符串进行自动解析）。
3.  点击 **创建连接**，然后点击 **启用** 即可切换查询源。

### 安全审计
所有操作均记录审计日志，可查询 `audit_logs` 表查看：
- 越权访问尝试（含 IP 地址）
- 导出操作记录
- 登录/登出记录
- 一次性链接使用记录

## ⚠️ 常见问题

**Q: 遇到 401 错误怎么办？**
A: 表示未登录或会话已过期，请重新登录。所有查询接口均需要认证。

**Q: 遇到 403 DATA_PERMISSION_DENIED 错误怎么办？**
A: 表示越权尝试访问非本人/本部门资产，该操作已被记录审计日志。请确认您查询的资产编码属于您的权限范围。

**Q: 遇到 403 EXPORT_PERMISSION_DENIED 错误怎么办？**
A: 导出功能仅资产管理员可使用，普通员工和外包人员无法导出。

**Q: 普通员工为什么看不到采购价格？**
A: 系统会根据角色自动过滤敏感字段，普通员工无法看到采购价格等财务相关信息。

**Q: 一次性链接可以多次使用吗？**
A: 不可以。一次性链接设计为单次使用，访问后立即标记为已使用，且 24 小时后自动过期。

**Q: 如何查看审计日志？**
A: 可直接查询数据库 `audit_logs` 表，记录了所有操作包括：登录、查询、导出、越权尝试、一次性链接使用等。

**Q: 如何重置数据库？**
A: 执行 `docker compose down -v` 删除数据卷，然后重新启动。
